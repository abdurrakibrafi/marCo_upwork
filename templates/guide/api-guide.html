🔐 Authentication Endpoints

🔑 Login: POST /api/auth/login/
Send email & password → Get access & refresh tokens
🔄 Refresh Token: POST /api/token/refresh/
Send refresh token → Get new access token (no re-login needed!)
📝 Register: POST /api/auth/register/
Create account → Get tokens immediately

🔄 Authentication Flow

Step 1: Login with credentials → Get access + refresh tokens
Step 2: Use Authorization: Bearer <access_token> for all API calls
Step 3: When access token expires → Use refresh endpoint
Step 4: Repeat Step 3 (no need to login again!) 🎉

📝 Standard Response Format

{ "success": true/false, "message": "Human-readable message", "timestamp": "2025-06-30T10:30:00Z", "status_code": 200, "data": {...}, // Only on success "error_code": "...", // Only on errors "errors": {...} // Only on validation errors }

⚠️ Error Codes (Use these instead of HTTP status codes)

VALIDATION_ERROR NOT_AUTHENTICATED AUTHENTICATION_FAILED PERMISSION_DENIED RESOURCE_NOT_FOUND RATE_LIMITED INTEGRITY_ERROR

💡 Pro Tips

  • ✅ Always check response.success first
  • ✅ Use error_code for programmatic error handling
  • ✅ Display message to users (it's human-friendly)
  • ✅ Store refresh token securely for auto-refresh
  • ✅ Use interactive docs below for testing
